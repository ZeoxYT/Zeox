# ZEOX — نشر على GitHub + Vercel

هذه حزمة جاهزة تحتوي على:
- `index.html` + `assets/` وباقي الملفات الثابتة → الواجهة الأمامية (الموقع نفسه).
- `api/index.mjs` → سيرفر الـ API (تشفير، رفع الملفات، توكنات) كـ Vercel Serverless Function واحدة تحتوي كل الكود مدمج بداخلها.
- `vercel.json` → يوجّه الطلبات: كل شي يبدأ بـ `/api/` أو `/files/` يروح للسيرفر، والباقي يروح للواجهة (index.html).

## سبب مشكلة الـ 404 سابقاً

كنت رافع الواجهة فقط بدون سيرفر API، فكل طلب على `/api/...` كان يرجع 404 لأن Vercel ما يلقى شي يرد عليه. هذي الحزمة فيها السيرفر جاهز بصيغة Serverless Function تشتغل تلقائياً على Vercel.

## خطوات النشر

### 1. ارفع الحزمة على GitHub
أنشئ مستودع (repository) جديد على GitHub وارفع محتويات هذا المجلد بالكامل (بما فيها مجلد `api/`).

### 2. أنشئ قاعدة بيانات Postgres خارجية (مجانية)
قاعدة البيانات الحالية داخل Replit **غير متاحة من خارج Replit**، فلازم قاعدة بيانات خارجية. الأسهل والمجاني:

- اذهب إلى [neon.tech](https://neon.tech) وأنشئ حساب مجاني.
- أنشئ مشروع جديد (Project) واختر أي اسم.
- من صفحة الاتصال (Connection Details)، انسخ رابط الاتصال (يبدأ بـ `postgresql://...`).

### 3. استورد المشروع في Vercel
- اذهب إلى [vercel.com](https://vercel.com) → New Project → اختر المستودع من GitHub.
- في إعدادات Environment Variables أضف:
  - `DATABASE_URL` = رابط قاعدة البيانات من Neon (الخطوة 2).
- اترك باقي الإعدادات الافتراضية (لا حاجة لـ Build Command، المشروع جاهز وثابت).
- اضغط Deploy.

### 4. أنشئ جداول قاعدة البيانات (مرة واحدة فقط)
بعد ربط `DATABASE_URL`، تحتاج تنشئ الجداول في قاعدة البيانات الجديدة. من جهازك (يتطلب Node.js مثبت):

```bash
git clone <رابط-المستودع-الخاص-بك>
cd <اسم-المستودع>
npm install drizzle-kit drizzle-orm pg dotenv
```

أو الأسهل: تواصل معي داخل Replit وأشغّل لك أمر `drizzle-kit push` مباشرة على رابط قاعدة بيانات Neon الجديدة (أرسل لي رابط الاتصال كـ Secret وليس في الشات).

### 5. تحقق
بعد اكتمال النشر، افتح رابط الموقع من Vercel وجرّب تسجيل الدخول، التشفير، الرفع، وإنشاء توكن — يجب تعمل كلها بدون 404.

## ملاحظات مهمة

- تسجيل الدخول (Google) يعمل عبر Firebase وهو مستقل تماماً عن مكان الاستضافة، فلن تحتاج أي إعداد إضافي له.
- إذا غيّرت أي كود مستقبلاً، تحتاج تعيد بناء المشروع من Replit وتصدير حزمة جديدة (نفس الطريقة)، لأن هذي حزمة "بناء نهائي" جاهزة وليست مصدر حي.
- خدمة التشفير (`goofyscator.lua.cz`) مستدعاة من السيرفر مباشرة (server-side)، تعمل تلقائياً على Vercel بدون أي إعداد إضافي.
